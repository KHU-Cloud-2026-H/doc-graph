# DocGraph 제품 명세

Notion 문서 간 의존 관계를 그래프로 모델링하고, 변경 전파와 정합성 위반을 자동 감지하는 SaaS.

## 사용자 역할

| 역할 | 권한 |
| --- | --- |
| Project Admin | 프로젝트 생성, Notion 연동, 문서 타입 편집, 룰 테이블 편집 (체크 항목 수정·커스텀 룰 추가·삭제) |
| Team Member | 그래프 조회, 정합성 위반 확인, 충돌 해소 |

## 기능

### F1. Notion 연동

- Notion OAuth를 통해 워크스페이스를 연결할 수 있어야 한다.
- 연결된 워크스페이스에서 최상위 폴더 하나를 프로젝트 단위로 등록할 수 있어야 한다.
- 등록된 폴더 하위 페이지에 변경이 발생하면 Notion Webhook으로 실시간 이벤트를 수신해야 한다.
- 이벤트 수신 후 Notion API를 호출해 변경된 페이지의 전체 내용을 조회할 수 있어야 한다.

### F2. 문서 타입 분류

- 동기화된 문서는 AI API를 호출해 문서 타입을 자동 분류해야 한다. 폴더명·제목에 타입 키워드가 명확히 포함된 경우 API 호출을 생략하고 직접 분류한다.
- 기본 제공 타입은 다음과 같다.
  - `meeting_notes`
  - `planning`
  - `requirements`
  - `design`
  - `research`
- Project Admin은 타입을 추가·삭제·수정할 수 있어야 한다.
- 사용자는 자동 분류된 결과를 수동으로 수정할 수 있어야 한다.

### F3. 의존 관계 그래프

- Notion 페이지 링크·멘션이 감지되면 해당 연결을 의존 관계로 자동 등록해야 한다.
- 등록된 의존 관계의 문서 타입 쌍이 룰 테이블에 해당하면, 해당 룰의 체크 항목으로 정합성 검증을 수행한다.
- 기본 제공 룰은 다음과 같다.

| 출발 | 도착 | 체크 항목 |
| --- | --- | --- |
| `meeting_notes` | `planning` | 결정사항 반영 여부 |
| `meeting_notes` | `requirements` | 결정사항 반영 여부 |
| `research` | `planning` | 전제 조건 유지 여부 |
| `planning` | `requirements` | 범위 일치 여부 |
| `requirements` | `design` | 스펙 일치 여부 |

- 사용자는 의존 관계를 수동으로 추가·삭제할 수 있어야 한다. 추가 시 룰 테이블 기반 또는 검색 기반 연결 추천을 제공해야 한다.
- 생성된 의존 관계를 시각적 그래프로 조회할 수 있어야 한다.
- 여러 프로젝트 간 전환이 가능해야 하며, 프로젝트별로 독립된 그래프를 조회할 수 있어야 한다.

### F4. 결정사항 추출 및 추적

- `meeting_notes` 타입 문서가 동기화되면 AI API를 호출해 결정사항 목록을 추출해야 한다.
- 추출된 결정사항은 출처 문서와 추출 시각을 함께 저장해야 한다.
- 사용자는 별도 뷰에서 결정사항 목록을 조회할 수 있어야 하며, 각 항목은 출처 문서 링크를 포함해야 한다.

### F5. 정합성 검증

- 문서 동기화 시 페이지 전체 내용을 스냅샷으로 저장해야 한다.
- Notion Webhook 수신 후 현재 내용과 스냅샷을 블록 단위로 비교해 변경 블록을 식별해야 한다.
- 변경 블록과 전후 맥락을 소스로, 의존 관계로 연결된 문서 전체를 타깃으로 AI 검증을 비동기 요청해야 한다.
- AI API를 non-blocking으로 호출하고, 타깃 문서의 충돌 구간, 원인, 수정 제안을 반환받아야 한다.
- 정합성 위반이 감지된 연결 엣지는 그래프에서 빨간색으로 표시되어야 한다.
- 충돌 구간은 인라인으로 표시되어야 하며, 수정 제안을 함께 제공해야 한다.
- 해당 Notion 페이지에는 경고 코멘트로 정합성 위반 내용을 자동 삽입해야 한다.

### F6. 알림

- 정합성 위반이 감지되면 그래프 뷰 및 대시보드 상단에 미해소 위반 항목 목록을 표시해야 한다.
- 위반 항목은 해소 완료 전까지 지속적으로 표시되어야 한다.
- 정합성 위반이 감지되면 프로젝트에 설정된 Webhook URL로 알림을 발송할 수 있어야 한다.
- Webhook 알림은 Slack, Discord 등 일반적인 Webhook 수신 서비스와 호환되어야 한다.
- Webhook URL은 프로젝트별로 설정할 수 있어야 한다.

### F7. 충돌 해소

- 사용자는 정합성 위반 항목을 `해소됨` 상태로 수동 마킹할 수 있어야 한다.
- 해소된 항목은 그래프에서 시각적으로 구분되어야 한다.
- 충돌이 감지된 문서 쌍은 이분할 비교 뷰로 조회할 수 있어야 한다.
- AI 수정 제안을 승인하면 해당 내용이 Notion 문서에 직접 반영되어야 한다.
- 사용자는 AI 수정 제안을 거부할 수 있어야 하며, 거부 시 위반 항목은 유지된다.
- 사용자는 비교 뷰에서 직접 내용을 수정해 Notion에 반영할 수 있어야 한다.

## 화면 구성

```text
랜딩
  ↓ (로그인)
Notion OAuth2
  ↓
워크스페이스 목록
  ├─ 워크스페이스 설정 (연동·Webhook 관리)
  ├─ 그래프 뷰
  │    └─ 문서 비교 패널 (충돌 엣지 클릭 시)
  └─ 대시보드
```

### 랜딩

- 비로그인 상태: 서비스 소개 콘텐츠 표시
- 로그인 상태: 워크스페이스 목록으로 자동 리다이렉트

### 로그인

- Notion 계정으로 OAuth2 인증
- 로그인 성공 시 워크스페이스 목록으로 이동
- 별도 회원가입 플로우 없음. 첫 OAuth 완료 시 Notion 사용자 정보로 DocGraph 계정을 자동 생성한다.

### 워크스페이스 목록

Notion OAuth로 연결된 계정이 보유한 워크스페이스 목록.

- **내 워크스페이스**: 내가 소유한 워크스페이스 목록
- **참여 중인 워크스페이스**: 팀원으로 초대받은 워크스페이스 목록
- 각 워크스페이스 클릭 시 그래프 뷰로 이동

**팀원 초대 (소유자 전용)**

- Notion 워크스페이스 멤버는 API로 목록을 조회해 선택 초대한다.
- 워크스페이스 외부인(Guest)은 Notion 계정 이메일로 초대한다.
- 초대받은 사용자가 해당 Notion 계정으로 DocGraph에 로그인하면 워크스페이스 접근 권한이 부여된다.

### 워크스페이스 설정

Notion 계정 연결 후 각 워크스페이스별 DocGraph 사용 설정을 관리하는 페이지.

> Notion 계정 자체는 OAuth 로그인 시 연결 완료. 이 페이지는 연결된 워크스페이스들을 DocGraph에서 어떻게 사용할지 설정하는 곳.

| 항목 | 설명 |
| --- | --- |
| 워크스페이스 추가 | Notion OAuth를 추가 실행해 새 워크스페이스를 연결한다. 기존 DocGraph 계정에 워크스페이스가 추가된다. |
| 정합성 관리 활성화 | 해당 워크스페이스의 문서 변경 감지 및 정합성 검증 활성화 여부 |
| Webhook URL | 충돌 감지 시 알림을 받을 Slack·Discord 등 Webhook URL |
| Webhook 연결 상태 | 설정된 Webhook의 정상 연결 여부 표시 |

### 그래프 뷰

워크스페이스 내 문서 의존 관계를 시각화하는 메인 화면.

- 문서를 노드, 의존 관계를 엣지로 표시
- 정합성 위반이 감지된 엣지는 빨간색으로 표시
- 충돌 엣지 클릭 시 문서 비교 패널 열림

**문서 비교 패널**

- 연결된 두 문서를 좌우 이분할로 표시 (git diff 스타일)
- 충돌 구간 인라인 하이라이트
- AI 수정 제안 표시
  - 승인: Notion 문서에 직접 반영
  - 거부: 제안 닫기, 위반 항목은 유지
  - 사용자가 패널 내에서 직접 수정 후 Notion에 반영
- 위반 해소 처리: 수동으로 `해소됨` 마킹 가능

### 대시보드

워크스페이스별 충돌 현황을 집계·기록하는 별도 페이지.

- 충돌 감지 건수 통계
- 충돌 발생·해소 이력 로그
- 미해소 위반 건수 집계
- 위반 발생·해소 이력 로그
- 기간 필터는 최근 30일 고정 (추후 확장 가능)

## 구현 제약

| 항목 | 제약 |
| --- | --- |
| Webhook 수신 응답 시간 | 3초 이내 응답 |
| AI 검증 타임아웃 | 단일 요청 30초 이내 |
| 충돌 해소 반영 | 사용자 마킹 즉시 그래프 반영 |
| idempotency | 동일 이벤트 재처리 시 Notion 코멘트 중복 삽입 금지 |
| 실패 격리 | 한 문서 쌍의 검증 실패가 다른 쌍에 영향을 주지 않아야 함 |
| 외부 연동 실패 폴백 | Webhook 알림·Notion 코멘트 삽입 실패 시에도 인앱 위반 표시 유지 |
| OAuth 토큰 보호 | 사용자 Notion OAuth 토큰은 암호화 저장 |
| 비밀정보 관리 | DB 비밀번호·AI API 키는 안전한 비밀 저장소로 관리 |

## 범위 제외

- 범용 인앱 문서 에디터 (충돌 해소 목적의 제한적 인라인 편집은 포함)
- GitHub 연동
- Notion 외 문서 소스 연동 (Confluence 등)
- Webhook 외 별도 동기화 방식 (주기적 폴링 등)
- 이메일 알림
